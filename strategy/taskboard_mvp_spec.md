# イマダルマ任務盤（MVP仕様）

> **判定（栄一・2026-07-14）**: いまの国家で **いちばん必要なアプリはこれ**（任務の見える化）。  
> **理由**: 営業が止まっている主因は「売り方」ではなく、緊急業務が見えず重要に手が伸びないこと。  
> **開発**: Claude Code / Codex で自社・簡易で可。ただし **MVP以外は作らない**。  
> **やさしい説明**: `daily_governance/ONE_SYSTEM.md`

---

## 1. 何を解決するか（成功の定義）

開けばすぐ分かること:

1. **今日やること**（少数）  
2. **緊急**（クレーム・期限切れ・本日期限）  
3. **重要だが急がない**（営業・投稿・大きな前進）— ここが毎日空いていると失敗  
4. **長期／いつか**  
5. **昨夜の振り返り → 翌日への繰り越し**

14日後に「何に追われているか分からない」が消えていれば成功。  
デザインの美しさ・通知の豪華さは成功条件に入れない。

## 2. 画面（これだけ）

| 画面 | 中身 |
|------|------|
| **ホーム＝四象限** | 緊急×重要の4マス＋件数。タップでタスク開く |
| **今日** | 今日やるものだけ（手動ピン or 「今日に入れる」）。上限 **7件**（超過は警告） |
| **全部一覧** | フィルタ: 期限／リスト／完了隐藏 |
| **振り返り** | 日付ごと。よかった／悪かった／翌日へ（手帳の夜の頁） |
| **新規** | タイトル・期限・緊急度(高/低)・重要度(高/低)・メモ。保存で象限に自動振り分け |

スマホでもPCでも同じURL（レスポンシブ）。ログイン不要（個人・ローカル優先可）。

### 四象限の自動振り分け

| 緊急 | 重要 | 象限名（表示） | 朝の扱い |
|------|------|----------------|----------|
| 高 | 高 | **今すぐ** | 最優先 |
| 高 | 低 | **雑務・急ぎ** | まとめて短時間 |
| 低 | 高 | **大事（防衛）** | 毎日最低1つ着手 |
| 低 | 低 | **後回し可** | 触らなくてよい |

長期タスク = 重要・低緊急、または期限が先のもの。  
クレーム = 緊急・高重要に入れる（タグ `claim` 任意）。

## 3. データ（最小）

タスク1件:

- id, title, dueDate(任意), urgency(high|low), importance(high|low), status(todo|doing|done), notes, createdAt, updatedAt  
- optional: tags[]（claim, pay, sales…）

振り返り1件:

- date, wentWell, wentWrong, carryToTomorrow[]（タスクidまたは文言）

保存先（どれか一つ・最初はA）:

- **A**: ブラウザ localStorage（いちばん早い・一人用）  
- **B**: このrepo内の `apps/taskboard/data.json`（Gitで履歴が残る）  

同期PC⇄携帯: 最初は **同じブラウザにログインしたChrome** か、Bなら Git。クラウド本格同期は **v2（後回し）**。

## 6. 運用との接続

| 起動後 | やること |
|--------|----------|
| 初日 | 頭・手帳・古いタスクを全部投入。緊急重要を振る |
| 毎朝 | 任務盤を開く → 今日≤7 → 「大事」から1つ確保 |
| 毎夕 | 振り返り1分 → 翌日へ繰越 |
| 尊徳 | 朝は任務盤のスクショ／要約をもとに today を短くする（二重管理しない） |

市販アプリ（To Do等）は **任務盤が動くまで**の応急で可。任務盤が動いたら本体を一本化。

---

## 7. 栄一の最終判断

- **必要性**: 最優先。ボトルネックはここ。  
- **自社開発**: 簡易MVPなら可（商品ではなく **自分の業務OSの道具**）。  
- **危険**: 「作り込む週」に入り、また緊急業務に埋もれること。だから **小さく出して使う**。
