# strategy — 戦略・売上レビュー領域

> **管轄**: AI栄一（戦略メンター）
>
> **承認**: 君主いまだ唯仁（最終決定）
>
> **実行接続**: AI尊徳（`daily_governance/`）

---

## この領域の目的

`strategy/` は、**AI栄一が信用・実業・商品・顧客・導線・売上戦略を設計し、週次・月次で見直すための領域**である。

第一国家目標「**2026年11月30日までに、サッカー以外の事業で月間総合売上50万円以上**」を、戦略レイヤーから支える。

AI尊徳が日々記録した実行結果をもとに、AI栄一が戦略を検証・更新する。

---

## AI尊徳との関係

```
AI栄一（strategy/）
  └── ロードマップ・商品・導線・やらない戦を設計
        ↓ 君主承認
AI尊徳（daily_governance/）
  └── 日次実行・進捗・内政・人格教育へ落とし込む
        ↓ 週末
AI尊徳（weekly_review.md）
  └── 実行材料をAI栄一へ渡す
        ↓
AI栄一（eiichi_weekly_review.md / eiichi_monthly_review.md）
  └── 戦略を見直し、来週・来月の実行方針を決める
```

| 観点 | AI栄一 | AI尊徳 |
|------|--------|--------|
| **問い** | 何を・なぜ・どの順で | 今日何を・いつ・どうやって |
| **時間軸** | 週次〜月次 | 日次〜週次 |
| **本領域** | `strategy/` | `daily_governance/` |
| **接続点** | 週次・月次レビュー | `weekly_review.md` で材料を渡す |

## 週次レビューと月次レビューの使い分け

| 種別 | ファイル | タイミング | 目的 |
|------|---------|-----------|------|
| **週次** | `eiichi_weekly_review.md` | 毎週末 | 乖離の早期検知、次週の一点突破、やらない戦の確認 |
| **月次** | `eiichi_monthly_review.md` | 毎月末 | ポートフォリオ見直し、捨てる戦・強化する戦、次月の売上仮説 |

**原則:**

- 週次は**微調整**。基本戦略は変えない
- 月次は**本格更新**。`roadmap_500k.md` のマイルストーン・ポートフォリオを見直す
- 50万円ポートフォリオ仮説は**固定ではなく毎週検証**するが、基本戦略（一点突破→二点展開）は壊さない

## 孫子兵法（補助原理）

AI栄一は渋沢栄一の実業精神を土台に、孫子兵法を**補助原理**として参照する。

| 原理 | 運用 |
|------|------|
| **戦わずして勝つ** | 既存資産・紹介・導線を優先。ゼロからの新規開拓を避ける |
| **やらない戦** | `no_battle_rules.md` で管理。勝てない戦ははっきり捨てる |
| **勝てない戦を避ける** | 週次レビューで確認。脱線パターンと接続 |
