`ai-imadaruma` は、自己統治論・仁子思想・じぶん憲法・四柱統治フレームワーク・明君七徳をもとに、**11月末までにサッカー以外の事業で月間総合売上50万円以上**を達成するための国家OSです。

---

## 2. 最高法規

* じぶん憲法 v3 を最高法規とする
* AI栄一、AI尊徳、四柱統治フレームワーク、戦略、日次運用はすべて憲法の下位に位置する
* 憲法と矛盾する場合は、憲法を優先する

---

## 3. 役割分担（混同禁止）

* AI栄一は戦略を担当する
* AI尊徳は実行を担当する
* AI栄一とAI尊徳を混同しない
* AI尊徳が戦略を勝手に変更しない
* AI栄一が日次実行を細かく抱え込みすぎない

---

## 4. 作業前ルール

すべての作業の前に、必ず次を実行すること。

```bash
git status
```

working tree が clean でない場合は、作業を開始せず、変更内容を報告すること。

---

## 5. 作業後ルール

作業後は必ず次を実行すること。

```bash
git status
git diff
```

必要に応じて対象範囲を限定して差分を確認すること。

---

## 6. コミット前ルール

コミット前に、必ず次を報告すること。

* 編集・作成したファイル
* 各ファイルの要点
* 旧表記チェック結果
* Git状態
* コミット可能かどうか

ユーザーの明示指示なしにコミットしないこと。

---

## 7. push前ルール

pushは、コミット後にユーザーが許可した場合のみ行うこと。

push後は必ず `git status` を確認し、リモートと同期していることを報告すること。

---

## 8. Phase分離ルール

* 複数のPhaseを混ぜない
* 戦略変更、憲法変更、日次運用変更、ファイル整理を同一コミットに混ぜない
* 1コミット = 1目的 を原則とする

---

## 9. 禁止事項（正式名称としての復活禁止）

以下を明確に禁止します。

* AI孫子、AI蕭何という旧名称を正式名称として復活させない
* 軍師、相国、統治総理AIという旧役職を正式役職として復活させない
* `archives/legacy/` を勝手に編集しない
* `cabinet/chief_statecraft_minister.md` を勝手に編集しない
* 正本DOCXを削除しない
* 旧版ファイルを勝手に削除しない
* 四柱そのものにAI主管を直接書かない
* AI主管定義を複数ファイルで再定義しない
* ロードマップの基本戦略を勝手に変更しない
* 収益源を勝手に増やしすぎない

許容される例：

* 変更履歴や移行説明など、文脈を明確にした「旧表記」参照は残してよい

---

## 10. 旧表記チェック

関連ファイルを編集した場合は、必要に応じて以下を実行すること。

```bash
grep -R "統治総理AI\|AI孫子\|AI蕭何\|軍師\|相国" .
```

ただし、次の例は「禁止事項」ではないため、文脈次第で残してよい。

* `archives/legacy/` や変更履歴・移行注記に残る旧表記（文脈が明確な場合）
* 補助原理としての単体表記 `孫子` / `蕭何`
