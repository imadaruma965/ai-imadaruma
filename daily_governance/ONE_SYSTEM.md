## いま困っていること（言い換え）

| 今さんの感覚 | かんたんな意味 |
|--------------|----------------|
| タスクに追われる | やることが多すぎて、どれから手を付けていいか分からない |
| 埋もれる | 大事なことが、急ぎの返信や雑務の下に隠れる |
| 取りこぼし | 書いた場所がいろいろで、見る場所を忘れる |
| 後回し | 期限があるのに、見えなくなって遅れる |
| ミス・クレーム | 遅れや抜けが、相手の不満になる |

だから必要なのは「新しい哲学」ではなく、**見る場所を一つにすること**。

## ベストな形（これを採用する）

```
【一台のアプリ】← 本物（ここがメイン）
  ・手帳の「今日・予定・期限」
  ・タスク帳の「やること一覧」
  ・携帯でもPCでも同じ画面
  ・日付を選んで入れられる
  ・全部のタスクが一覧できる

【今日だけメモ】（任意）
  ・朝、アプリから「今日これやる」を3〜5個だけ控える
  ・国家OSの today.md でも、アプリの「今日」リストでもよい
  ・二か所に全部は書かない
```

Markdownの表（`ops_commitments.md` など）は、アプリが決まるまでの下書き、または週1のまとめ用。  
**毎日の本体はアプリ一台。**

---

## アプリに入れるもの（これだけ分けておく）

リスト（箱）を作る。名前は日本語でよい。

| リスト名（例） | 入れるもの |
|----------------|------------|
| **今日・期限** | 締め切りがある仕事（支払い・発注・納品・返信） |
| **クレーム・やり直し** | 怒っている案件・やり直し（最優先） |
| **大事・自分から進める** | 営業・投稿・大きな仕事の一歩（急ぎじゃないが重要） |
| **予定** | 面談・収録・通院など日時が決まったもの |
| **待ち** | 相手の返事待ち（自分は触らない） |

日記代わりに続けるなら、各タスクの「メモ」欄に一言でよい。別の日記アプリは増やさない。

---

## 毎日の使い方（3手順）

1. **入ってきたらすぐアプリに1件**（LINEのあとでも）。頭に残さない  
2. **朝**: アプリで「期限切れ・今日・クレーム」を見る → 今日やるものを画面に出す  
3. **夜**: 終わったものにチェック。残ったものは日付を伸ばすか、明日に残す  

これで「手帳＋タスク帳＋途中のメモ」を一台にまとめられる。

## パソコンでも携帯でも、という条件

採用するアプリは次を満たすこと。

- スマホアプリがある  
- PC（ブラウザかアプリ）で同じリストが見える  
- 期限・日付が付けられる  
- 一覧ができる  

むずかしい比較はしない。候補はこの中から **今夜一つ**:

| 候補 | むいている人 |
|------|----------------|
| **Microsoft To Do** | PCもスマホも使う人（いちばん無難） |
| **TickTick** | カレンダーっぽく全部見たい人 |
| **Google タスク** | いつもGoogleカレンダーの人 |
| **Appleリマインダー** | iPhone中心で、PCはあまり使わない人 |

選んだら下に書いて14日変えない。中身を全部入れる。
