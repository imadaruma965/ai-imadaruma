# impulse_protocol — 脱線・衝動復帰プロトコル

> 衝動、脱線、逃避、過剰な調べ物が出たときの即時対応手順。
> **自責ではなく復帰を優先する。** 責めるためのものではない。

---

## 運用ルール

| 項目 | 内容 |
|------|------|
| **担当** | AI尊徳が案内。君主が実行 |
| **所要時間** | 3分以内 |
| **原則** | 止める → 記録する → 最小行動で戻る |
| **補助原理** | 蕭何 — 兵站を乱さず、記録を残し、継続を守る |

衝動の内容は `parking_lot.md` に記録して後で処理してよい。
プロトコル完了後、**主戦場または次の15分タスク**に戻る。

---

> これは失敗ではない。統治の正常なプロセスである。

---

### 2. どの衝動か（10秒）

- ⬜ **探索衝動** — 調べ物、情報収集、新しいアイデア、AIツール
- ⬜ **回避衝動** — 難しいタスク・売上行動を避けたい
- ⬜ **快楽衝動** — SNS、動画、ポルノ、娯楽
- ⬜ **完璧衝動** — 細部にこだわりすぎ、出せない
- ⬜ **他者衝動** — 依頼・誘いに引っ張られている
- ⬜ **構想衝動** — 設計・整備だけで満足し、実行しない

---

### 4. 3分復帰手順

| 秒数 | 行動 |
|------|------|
| **0〜30秒** | 衝動を1行で記録する |
| **30〜60秒** | 衝動の種類を選ぶ（上記） |
| **60〜120秒** | AI尊徳の問いに答える |
| **120〜180秒** | 最小復帰行動を1つだけ実行する |

---

### 5. AI尊徳の問い

> 積小為大 — 大きく戻ろうとしない。次の一手だけ考える。

- 今日の主戦場は何だったか？
- 今やるべき最小の15分タスクは何か？
- これは「今日やらないこと」に入っていないか？
- 戦略変更が必要か、それとも単なる逃避か？

### 7. 記録欄


## 衝動の種類別対処

| 種類 | 対処 |
|------|------|
| 探索衝動 | `parking_lot.md` に記録 → 主戦場に戻る |
| 回避衝動 | タスクを15分に分割 → 最初の15分だけやる |
| 快楽衝動 | タイマー5分だけ許可 → 終わったら主戦場に戻る |
| 完璧衝動 | 「80%で出す」と宣言 → 次の15分に進む |
| 他者衝動 | 「今日やらないこと」に追加 → 丁寧にお断り or 後日対応 |
| 構想衝動 | 構想は1行でメモ → 売上直結タスクに即復帰 |

## エスカレーション

| 状況 | 対応 |
|------|------|
| 1日5回以上の衝動 | 夜レビューで原因分析。AI尊徳が報告 |
| 同じ種類が3日連続 | 週末レビューでAI栄一へ戦略相談を検討 |
| 復帰できない | AI尊徳に「次の15分タスクを1つだけ提案して」と依頼 |

---

## AI尊徳への起動プロンプト

```
AI尊徳、脱線しました。impulse_protocol.md に従い、
自責せずに復帰手順を案内してください。
次の15分タスクを1つだけ提案してください。
```

---

## 注意

- 衝動の中に良いアイデアがあることもある → `parking_lot.md` で後日評価
- 復帰できなかった日も、夜レビューで責めない。明日の最初の15分に再配置する
