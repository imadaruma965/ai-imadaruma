# daily_governance — 日次統治領域

> **管轄**: AI尊徳（実行マネージャー）
>
> **承認**: 君主いまだ唯仁（最終決定・実行）
>
> **戦略接続**: AI栄一（`strategy/roadmap_500k.md`）

---

## この領域の目的

`daily_governance/` は、**AI尊徳が日々の実行・内政管理・人格教育を行うための運用領域**である。

第一国家目標「**2026年11月30日までに、サッカー以外の事業で月間総合売上50万円以上**」を、君主いまだ唯仁が毎日実行するための兵站・記録・継続運用の拠点である。

AI栄一が設計した戦略を、ここで日々の行動に落とし込む。

---

## 運用サイクル

```
朝   → morning_cabinet.md   AI尊徳による朝の閣議
日中 → today.md             実行・進捗・ログ
脱線 → impulse_protocol.md  衝動・逃避からの復帰
夜   → night_review.md      AI尊徳による統治レビュー
週末 → weekly_review.md     実行振り返り → AI栄一の戦略レビューへ接続
```

| 時間帯 | ファイル | 担当 | 所要時間 |
|--------|---------|------|---------|
| **朝** | `morning_cabinet.md` | AI尊徳 | 10〜15分 |
| **日中** | `today.md` | 君主（AI尊徳が整備） | 随時 |
| **脱線時** | `impulse_protocol.md` | AI尊徳 | 3分以内 |
| **夜** | `night_review.md` | AI尊徳 | 5〜10分 |
| **週末** | `weekly_review.md` | AI尊徳 → AI栄一 | 15〜30分 |

## AI尊徳とAI栄一の関係

```
AI栄一（戦略メンター）
  └── ロードマップ・商品・導線・やらない戦を設計
        ↓ 君主承認
AI尊徳（実行マネージャー）
  └── 日次タスク・進捗・内政・人格教育へ落とし込む
        ↓ 君主実行
週末レビュー
  └── 実行結果をAI栄一に渡し、戦略を見直す
```

| 観点 | AI栄一 | AI尊徳 |
|------|--------|--------|
| **問い** | 何を・なぜ・どの順で | 今日何を・いつ・どうやって |
| **時間軸** | 週次〜月次 | 日次〜週次 |
| **本領域** | `strategy/` | `daily_governance/` |
| **接続点** | 週末の戦略レビュー | 週末レビューで材料を渡す |

AI尊徳は戦略を勝手に変更しない。変更が必要なときは、週末レビューまたは臨時でAI栄一へエスカレーションする。

## 四柱統治フレームワークとの関係

日次運用では、**四柱**と**6領域**の二層で確認する。

| 層 | 内容 | 確認タイミング |
|----|------|--------------|
| **四柱** | 人格・内政・外交・財政の最低行動 | 朝閣議・夜レビュー |
| **6領域** | PUB / COM / COMC / EDU / TRE / INT | 朝閣議・today.md |

四柱の定義は `constitution/four_pillars.md` を正とする。AI主管は `cabinet/ai_jurisdiction.md` に分離されている。
