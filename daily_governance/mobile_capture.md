# 外出先・モバイル捕獲（LINE／受発注／連絡）

> **病**: PCから離れているとき、受発注・連絡・LINEが「後で」になり、忘れ・遅延になる。  
> **治し方**: 外出中は **処理しない／捕獲する**。処理は定時の短い窓で行う。  
> **接続先**: 捕獲メモ →（帰宅・定時）→ `ops_commitments.md` → 朝 `today.md`  
> **管轄**: 尊徳（習慣）／栄一（型）／君主（実行）  
> **上位**: `task_system.md`

---

## 結論（先に）

外出中に完璧な返信や発注を済ませようとするな。  
やることは二つだけである。

1. **60秒以内にできる一手**（スタンプ・「確認して今日中に返す」／金額確定の短いOK）だけ送る  
2. **できないことは1行捕獲**して、頭と通知から降ろす  

捕獲先は **スマホの器を一つだけ** に固定する（下記）。PCの台帳は夜または定時で同期する。

---

## 道具（一つ選んで固定）

| 優先 | 器 | 使い方 |
|------|-----|--------|
| **推奨A** | スマホのメモ（Appleメモ／Google Keep 等）ノート名固定: `IMADA_INBOX` | 1行追加だけ |
| **推奨B** | LINEの「自分用ノート／Keep／自分宛メモ」 | 現場がLINE中心ならB |
| **不可** | 頭の中・「後で開く」未読の山・複数メモアプリ並行 | |

選んだらこのファイルの一行に書いてロックする。

**今さんの固定捕獲先**: `_______________`（君主が記入）  
※ 義務アプリを採用したら、ここを「**（アプリ名）に直接1件追加**」と書いてよい。別メモINBOXは不要になることが多い。

PC側の受け皿: `ops_commitments.md` の「モバイルから降りたもの」。

---

## LINEが来たとき（現場プロトコル・60秒）

```
通知を見る
  ├─ 60秒で返せる → 返す → 終わり
  └─ 返せない／調べが要る／金額・納期判断
        → 既読にしてもよいが「後で」だけにしない
        → IMADA_INBOX に1行書く
        → 相手に短い仮返答（任意・信用が要る相手は推奨）
```

### 1行の型（これ以外書かない）

```
[日付時刻] 誰: 何が欲しい／自分の次の一手 / 期限仮（今日夜・明日・金曜など）
例: 7/14 14:20 A社B: 見積催促 → 金額確認して返信 / 今夜
例: 7/14 15:01 発注C: 素材手配依頼 → PO台帳へ / 明日昼
```

### 仮返答の定型（コピペ用）

- 「確認して、今日中（または明日午前）に返す。」  
- 「受領した。手配内容を確認して折り返す。」  
- 「いま移動中のため、詳しくは ○○ までに返す。」  

「後で見る」は返答ではない。仮返答か1行捕獲のどちらか必须。

## 受発注が来たとき（外出中）

| 段階 | 外出中にやること | やらないこと |
|------|------------------|--------------|
| 受領 | 仮返答＋INBOX1行（金額・品名・相手） | 完璧な発注書作成 |
| 判断が要る | 「今夜／明日に確定」と期限を1行に書く | 歩いたまま仕様を詰めない |
| 即決できる少額 | その場でOK → INBOXに「済：内容」 | 済を書かず記憶に残す |

種類タグ（書ければ）: `PO` `PAY` `REP` `SCH`

---

## 定時処理窓（ここが「改善」の本体）

外出中の敗北は「窓がない」ことである。1日 **2回まで** 処理する（それで足りる）。

| 窓 | 目安 | やること（各5〜10分） |
|----|------|----------------------|
| **昼** | 食事前後 | LINE未読のうち業務だけざっと見る。返せるもの返す。INBOX追記 |
| **夕〜夜** | 帰宅後〜夜レビュー前 | **INBOX → `ops_commitments.md` へ全移動**。空にする。未返信は REP で期限付き |

窓以外の通知は、原則 **捕獲のみ**（深い返信・見積・発注手続きは窓で）。

尊徳チェック: 夜の時点で `IMADA_INBOX` が空か。空でなければ tonight の最低行動に「INBOXゼロ」を入れる。

---

## PCへの降ろし方（夜・5分）

1. スマホ INBOX を上から読む  
2. 各行を `ops_commitments.md` の適切な段（本日期限／3日以内／今週〜今月）へ転記  
3. 種類を付す（PAY/PO/REP…）  
4. スマホ行を消す、または「↓済」  
5. 翌朝、義務スキャンで today へ最大2件  

これが回り始めれば、「外出中に忘れた」は **台帳漏れ** として見えるようになる（感覚ではなく）。

---

## 段階

| 段階 | 内容 |
|------|------|
| **0（今週）** | 捕獲先を1つ決める。LINE来たら1行か仮返答。夜にINBOXゼロを目指す |
| **1** | 昼・夕の処理窓をカレンダー／アラームに固定 |
| **2** | 未返信が48時間超えたら台帳「期限切れ」へ強制転記（信用ルール） |
| **3** | どうしても漏れる相手だけ通知の例外（重要チャットのピン留め等）。全体の自動化はしない |

---

## やらないこと

- 外出中に「全部返信してから移動する」完璧主義  
- 未読放置をメモリにする  
- 新しいタスクアプリ探索（INBOXが一つ増えるだけ）  
- LINEを常時監視（注意力の分度破り）
