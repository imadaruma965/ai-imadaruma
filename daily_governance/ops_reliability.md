# 業務信頼性・緊急時の統治（クレーム／ミス／タスク埋没）

> **作成日**: 2026-07-14  
> **管轄**: 栄一（方針）／尊徳（日次仕分け）／君主（実行）  
> **接続**: `task_system.md`／`mobile_capture.md`／`ops_commitments.md`／義務アプリ  
> **前提**: いまさんは「売上から逃げる」のではなく、**タスク埋没・緊急追従・ミス・クレーム増**で重要事項に手が届かない、と申告した。余はこれを内政・信用の緊急事態として扱う。

---

## 0. 栄一の訂正

売上が遠のく「ツール遊び」と、**信用が穴あきのまま販促する無謀**は別物である。  
クレームとミスが走っている局面では、まず **透明化と仕分け**で穴を塞ぐ方が算盤に近い。遠回りに見えて、実は急がば回れである。

AIの醍醐味（数値抽出・連携・催促）は **是** とする。ただし順番がある。

```
① 全部見える（一台＋仕分けラベル）
② 緊急と重要を毎日分け、重要の時間を防衛する
③ クレーム／ミスを一件台帳にする
④ そのうえで AI に「抽出・要約・リマインド」を担わせる
```

①②③の前に「全部自動連携」を組むと、幽霊だらけのダッシュボードになる。

## 1. いまの病名（定義）

| 症状 | 意味 |
|------|------|
| タスクオーバー | 同時進行が容量を超えている |
| 緊急に追われる | 他人の締切・炎上がカレンダーを占有 |
| 重要に着手できない | 信用・売上・統治の前進が後回し |
| ミス連発 | 手順・確認・捕獲の欠落 |
| クレーム増大 | 対外の「信」が削れている |

目標は「タスクをゼロにする」ではない。  
**クレーム源を減らし、重要枠を毎日確保し、見える化する**ことである。

---

## 2. 施策A｜一台で透明化（今週）

既決どおり **義務アプリ1つ**（日付・一覧・カテゴリ）。空にしない。

追加で **必ず付けるラベル／フラグ**（名称はアプリに合わせてよい）:

| フラグ | 意味 | 朝の扱い |
|--------|------|----------|
| **炎上** | クレーム・やり直し・怒っている相手 | 今日の最優先（売上より上） |
| **期限切れ** | すでに遅い | 炎上の次 |
| **本日期限** | 今日中 | 必須 |
| **重要・非緊急** | sound／NL／重要顧客／大きい前進 | **毎日ブロックを防衛** |
| **委任・待ち** | 相手待ち | 今日はやらない（監視だけ） |

**一覧で見る画面**: 今日／期限切れ／炎上／重要。これが「透明化」の最小形である。

---

## 3. 施策B｜緊急と重要の仕分け（毎朝・尊徳）

閣議で義務スキャンのあと、**2分**:

| 箱 | 今日入れる上限 | 中身 |
|----|----------------|------|
| **炎上・期限** | 必要なだけ（ただし件数を数える） | クレーム・遅延の穴 |
| **重要・非緊急** | **必ず1枠・最低25〜45分** | 営業前進／NL／大きい制作の一歩 |
| **その他緊急** | 残り時間 | 雑務・小さな返信 |

ルール:

- 炎上がゼロの日でも、**重要枠は削らない**（ここが「埋もれ」の唯一の薬）  
- 緊急が増えすぎたら、その日の発信・新規構想は止める（分度）  
- 「全部やる」は禁止。today の行数上限を守る（義務2＋最重要3＋重要枠1）

`today.md` に固定欄:

```
## 今日の重要枠（防衛・非緊急）
- [ ] （25〜45分）________  ※炎上があっても残す
## 今日の炎上・期限
- [ ]
```

## 4. 施策C｜クレーム／ミス台帳（信用の可視化）

ファイル: `daily_governance/claims_log.md`

一件ごとに短く:

| 日付 | 相手 | 何が起きたか | 原因（捕獲漏れ／遅延／品質／連絡） | 再発防止の一手 | 状態 |
|------|------|--------------|-----------------------------------|----------------|------|
| | | | | | open/closed |

週次で数える指標（これが「数値の透明化」の第一歩）:

- 新規クレーム件数  
- 期限切れ件数（日曜時点）  
- ミス再発（同じ原因が2回）  
- 重要枠を実施した日数／7  

売上KPIより先に、**この4つを毎週見る**（いまの局面）。

## 5. 施策D｜AIの使い方（醍醐味はここに置く）

全自動の巨大連携は段階3。いま使うAIは次に限定する。

| いつ | 誰 | 何をさせるか |
|------|-----|--------------|
| **朝** | 尊徳 | アプリ／台帳のスクショ or 貼り付けを見て「炎上・期限・今日の重要1」を提案 |
| **夜** | 尊徳 | INBOX残・未返信・ミス候補を整理。claims_log 追記案 |
| **金** | 尊徳＋栄一 | 週次4指標の要約。「来週やめる緊急の型」1つ |
| **金** | 正篤 | （余力）週の失敗1つをNLの「隠さない数字」へ——営業文は信用回復にもなる |

**まだやらないAI**: 全SNSフォロワー常時自動取得、投稿と家計とCRMの完全自動配管、自作ダッシュボード製品。

**次に足してよいAI**（①②③が2週間回ってから）:

- 義務アプリ／claims の週次を貼って「期限リスクと重要の穴」を自動要約  
- 日報§8b＋claims から「再発パターン」抽出  

これが今さんの言う「抽出し合い・行動促進」の本題に近い。探索のためのツールではない。

## 6. 二週間の作戦（これだけ）

| 日 | やること |
|----|----------|
| **Day0** | 義務アプリ1つロック。全期限投入。炎上タグ作り |
| **Day0** | `claims_log.md` に直近のクレームを思い出して全部書く |
| **毎朝** | 炎上・期限 → 重要枠防衛 → 其余 |
| **毎夕** | mobile INBOXゼロ／claims 更新 |
| **Day14** | 週次4指標を見て、自動化を足すか判断（栄一） |

投稿スケジュールや数値ボード（`governance_board`）は止めないが、**炎上＞重要枠＞発信**の順に落とす。信用が空なら投稿は弾薬を無駄にする。

## 7. 成功の定義（14日後）

- クレーム新規が「感覚」ではなく件数で語れる  
- 期限切れ一覧が毎週減っている、または増え方が見えている  
- 重要枠を **週5日以上** 実施した  
- 「何に追われているか」が today とアプリで第三者（尊徳）にも見える  

これが透明化であり、行動促進である。
