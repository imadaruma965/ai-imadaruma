# night_review — 夜の統治レビュー

> 1日の終わりに、AI尊徳と君主が行う統治レビュー。
> 完了後、内容を `archives/daily_logs/YYYY-MM-DD.md` にコピー保存する。

---

## 運用ルール


| 項目          | 内容                            |
| ----------- | ----------------------------- |
| **所要時間**    | 5〜10分                         |
| **実施タイミング** | 1日の統治行為の最後                    |
| **担当**      | AI尊徳が整理、君主が承認                 |
| **原則**      | できなかったことは責めない。明日の最初の15分に再配置する |
| **補助原理**    | 蕭何 — 記録を残し、兵站を整え、明日へ継続する      |


振り返り後、`today.md` をリセットする（翌朝の閣議で再構築）。

## アーカイブ手順

1. このファイルの内容を `archives/daily_logs/YYYY-MM-DD.md` にコピー
2. `today.md` の内容をクリア（テンプレート状態に戻す）
3. `impulse_protocol.md` の衝動ログをクリア
4. 翌朝 `morning_cabinet.md` で新しい閣議を開始

---



## 3日ルール

同じタスクが3日連続で繰り越されている場合:

- AI尊徳は原因を分析する
- タスクの分割、領域の変更、または「やらないこと」への移動を提案する
- 戦略の問題なら、週末にAI栄一へエスカレーションを検討する
- 君主の判断を仰ぐ

---
